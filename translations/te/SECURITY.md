# భద్రత

Microsoft మా సాఫ్ట్‌వేర్ ఉత్పత్తులు మరియు సేవల భద్రతను తీవ్రంగా పరిగణిస్తుంది, ఇందులో [Microsoft](https://github.com/Microsoft), [Azure](https://github.com/Azure), [DotNet](https://github.com/dotnet), [AspNet](https://github.com/aspnet) మరియు [Xamarin](https://github.com/xamarin)తో సహా మా GitHub సంస్థల ద్వారా నిర్వహించబడే అన్ని సోర్స్ కోడ్ రిపోజిటరీలు ఉన్నాయి.

మీరు [Microsoft యొక్క భద్రతా బలహీనత నిర్వచనం](https://aka.ms/security.md/definition)కు అనుగుణంగా ఉండే Microsoft యాజమాన్యంలోని ఏదైనా రిపోజిటరీలో భద్రతా బలహీనతను కనుగొన్నారని మీరు విశ్వసిస్తే, దయచేసి దిగువ వివరించిన విధంగా దానిని మాకు నివేదించండి.

## భద్రతా సమస్యలను నివేదించడం

**దయచేసి పబ్లిక్ GitHub సమస్యల ద్వారా భద్రతా బలహీనతలను నివేదించవద్దు.**

బదులుగా, దయచేసి వాటిను [https://msrc.microsoft.com/create-report](https://aka.ms/security.md/msrc/create-report) వద్ద Microsoft Security Response Center (MSRC)కి నివేదించండి.

మీరు లాగిన్ చేయకుండా సమర్పించడానికి ఇష్టపడితే, [secure@microsoft.com](mailto:secure@microsoft.com)కి ఇమెయిల్ పంపండి. సాధ్యమైతే, మీ సందేశాన్ని మా PGP కీతో ఎన్‌క్రిప్ట్ చేయండి; దయచేసి దానిని [Microsoft Security Response Center PGP Key page](https://aka.ms/security.md/msrc/pgp) నుండి డౌన్‌లోడ్ చేసుకోండి.

మీరు 24 గంటల్లో ప్రతిస్పందన పొందాలి. ఏదైనా కారణంగా మీరు చేయకపోతే, మేము మీ అసలు సందేశాన్ని అందుకున్నామని నిర్ధారించుకోవడానికి దయచేసి ఇమెయిల్ ద్వారా ఫాలో-అప్ చేయండి. అదనపు సమాచారం [microsoft.com/msrc](https://www.microsoft.com/msrc)లో చూడవచ్చు.

సాధ్యమైన సమస్య యొక్క స్వభావం మరియు పరిధిని బాగా అర్థం చేసుకోవడంలో మాకు సహాయపడటానికి దయచేసి దిగువ జాబితా చేయబడిన అభ్యర్థించిన సమాచారాన్ని (మీరు అందించగలిగినంత) చేర్చండి:

* సమస్య రకం (ఉదా. buffer overflow, SQL injection, cross-site scripting, మొదలైనవి)
* సమస్య యొక్క అభివ్యక్తికి సంబంధించిన సోర్స్ ఫైల్(ల) పూర్తి మార్గాలు
* ప్రభావిత సోర్స్ కోడ్ యొక్క స్థానం (tag/branch/commit లేదా ప్రత్యక్ష URL)
* సమస్యను పునరుత్పత్తి చేయడానికి అవసరమైన ఏదైనా ప్రత్యేక కాన్ఫిగరేషన్
* సమస్యను పునరుత్పత్తి చేయడానికి దశల వారీ సూచనలు
* Proof-of-concept లేదా exploit కోడ్ (సాధ్యమైతే)
* సమస్య యొక్క ప్రభావం, దాడి చేసే వ్యక్తి సమస్యను ఎలా ఉపయోగించుకోవచ్చో సహా

ఈ సమాచారం మీ నివేదికను మరింత త్వరగా ట్రయాజ్ చేయడంలో మాకు సహాయపడుతుంది.

మీరు బగ్ బౌంటీ కోసం నివేదించినట్లయితే, మరింత పూర్తి నివేదికలు అధిక బౌంటీ అవార్డుకు దోహదపడతాయి. మా క్రియాశీల ప్రోగ్రామ్‌ల గురించి మరిన్ని వివరాల కోసం దయచేసి మా [Microsoft Bug Bounty Program](https://aka.ms/security.md/msrc/bounty) పేజీని సందర్శించండి.

## ఇష్టపడే భాషలు

మేము అన్ని కమ్యూనికేషన్‌లను ఆంగ్లంలో ఉండాలని ఇష్టపడతాము.

## విధానం

Microsoft [Coordinated Vulnerability Disclosure](https://aka.ms/security.md/cvd) సూత్రాన్ని అనుసరిస్తుంది.
